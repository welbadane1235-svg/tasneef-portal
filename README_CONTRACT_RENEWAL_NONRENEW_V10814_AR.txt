سبب خطأ operator does not exist: text = bigint
----------------------------------------------
مصدر الخطأ الثابت في النسخة السابقة هو RPC:
public.save_contracts_services_editor_v10807

كان تعريف المعامل:
p_project_id bigint

بينما بعض الجداول المنشأة أو القديمة قد تحتوي project_id من نوع text. وكانت الدالة تستخدم مقارنات مباشرة مثل:
- projects.id = p_project_id
- project_contract_smart.project_id = p_project_id
- contract_services.project_id = p_project_id

إذا كان أحد الأعمدة text والمعامل bigint، يصدر PostgreSQL الخطأ 42883:
operator does not exist: text = bigint

الحل في V10814
---------------
1. حدود RPC الجديدة تستقبل المعرفات القادمة من الواجهة كـ text.
2. تتم المقارنة بصورة صريحة وآمنة:
   id::text = p_id
   project_id::text = p_project_id
3. ملف SQL يفحص النوع الفعلي لـ projects.id بواسطة pg_catalog.
4. يحاول توحيد project_id في الجداول التابعة إلى النوع الحقيقي نفسه.
5. إذا وجد قيمة غير قابلة للتحويل، لا يحذفها ولا يخمن ربطها؛ يسجلها في:
   public.contract_identifier_issues_v10814
6. يسجل نتيجة كل جدول وعدد السجلات في:
   public.contract_identifier_migration_audit_v10814

آلية «لم يجدد»
----------------
- نافذة تأكيد ذكية داخل نفس الصفحة، ولا تستخدم alert/prompt/confirm.
- السبب إلزامي، و«سبب آخر» يتطلب شرحًا.
- RPC واحدة تنفذ العملية داخل Transaction قاعدة البيانات:
  - العقد: not_renewed وis_active=false.
  - المشروع: is_active=false وstatus=stopped.
  - stopped_reason=contract_not_renewed.
  - التوزيعات الحالية: end_date=current_date وis_active=false وstatus=ended وend_reason=project_contract_not_renewed، حسب الأعمدة الموجودة.
  - monthly_distribution: يقتصر الإنهاء على الشهر الحالي والمستقبل عند وجود month_key.
  - الخدمات المتبقية المستقبلية المرتبطة بالعقد: cancelled_due_to_non_renewal.
  - التنبيه: resolved / not_renewed.
  - سجل التعديلات: العقد، المشروع، التوزيعات، الخدمات والتنبيه.
- لا يتم إيقاف العامل أو المشرف نفسه.
- لا يستخدم DELETE لأي عقد أو خدمة أو مرفق أو توزيع تاريخي.
